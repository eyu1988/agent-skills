# process-daily

## 路径常量

- 日捕获目录：`{{CAPTURE_DIR}}`
- 知识库根目录：`{{KNOWLEDGE_DIR}}`

## 触发方式

- `处理今天`
- `/process-daily`

## 执行步骤

### 第一步：定位今日文件

```bash
TODAY=$(date +%Y-%m-%d)
FILE="{{CAPTURE_DIR}}/$TODAY.md"
[ ! -f "$FILE" ] && echo "今天没有日捕获文件，请先用 /record 添加内容。" && exit 0
```

### 第二步：读取并过滤

读取今日文件，逐条过滤"📥 今日捕获"区域的内容。

**保留标准（满足任一即保留）：**
1. 改变了原有认知——和之前理解不一样的东西
2. 将来会反复用到——这是第二次或以上遇到这个概念

**其余全部标记丢弃。**

### 第三步：分类并写入知识库

对每条保留内容，判断应归入哪个文件（路径相对于 `{{KNOWLEDGE_DIR}}`）：

| 内容类型 | 目标文件 |
|---------|---------|
| CLI、工具用法 | `AI_Coding_知识库/07_AI_Coding_学习体系/01_CLI.md` |
| AI Coding 架构、技术地图 | `AI_Coding_知识库/07_AI_Coding_学习体系/02_AICoding技术地图.md` |
| ralph-loop、omh 相关 | `AI_Coding_知识库/07_AI_Coding_学习体系/03_ralph-loop-omh演进.md` |
| Agent、编排、Base 概念 | `AI_Coding_知识库/07_AI_Coding_学习体系/04_AI_Coding_Base认知.md` |
| 工作流、信息管理 | `工作信息处理流.md` |
| 其他技术概念 | `AI_Coding_知识库/` 下最匹配的文件，若无则新建 |

写入格式（追加到对应文件末尾）：

```
## YYYY-MM-DD 补充

- [提炼后的要点，一句话]
```

若内容较多或改变了文件的核心观点，直接更新正文对应章节，不追加。

### 第四步：回填处理结果

更新今日文件的"✅ 处理结果"区域：

```markdown
## ✅ 处理结果

**保留并入库（N 条）：**
- [条目摘要] → [目标文件名]

**丢弃（N 条）：**
- [条目摘要]（原因：不满足保留标准）

**今日认知偏移：**
- [若有，一句话总结]
```

### 第五步：报告

输出简短报告：
- 处理了多少条
- 写入了哪些文件
- 有没有今日认知偏移
