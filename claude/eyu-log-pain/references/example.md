# 典型示例

## 示例：FINC 域消耗型痛点，选择深入分析

**用户输入**

```
录入痛点：每次看到好机会就冲动下单，事后后悔，但下次还是控制不住
```

---

**第一步：解析痛点**

- 域：`FINC`（理财投资）
- 严重程度：🟡 消耗型——不影响正常生活，但持续消耗精力和金钱

---

**第二步：生成 ID**

读取登记本，今日 FINC 已有 1 条，本条序号为 002。

ID：`FINC-20260520-002`

---

**第三步：5Why 根因分析**

1. 为什么冲动下单？→ 看到机会时情绪激动，判断力降低
2. 为什么情绪会激动？→ 没有冷静期缓冲，看到就决策
3. 为什么没有冷静期？→ 缺少"下单前必须完成"的强制检查流程
4. 为什么没有建立检查流程？→ 以前觉得自己能靠意志力控制
5. 根因：**依赖意志力而非流程约束**，缺少下单前的强制摩擦机制

---

**第四步：写入登记本**

追加到 `{{PAIN_DIR}}/01-痛点登记本.md` 🟡 消耗型分区：

```
#### FINC-20260520-002 · 冲动下单难以自控 · 2026-05-20

| 字段 | 内容 |
|------|------|
| **域** | FINC |
| **严重程度** | 🟡 消耗型 |
| **状态** | 🆕 新增 |
| **登记日期** | 2026-05-20 |

**痛点描述**：看到投资机会时情绪驱动下单，缺乏冷静期，事后频繁后悔，但行为模式未改变。

**根因（5Why）**：
依赖意志力而非流程约束；缺少下单前的强制摩擦机制，导致情绪高峰时直接决策。

**解决方案**：→ 待分析
```

同时将 FINC 域"活跃"计数 +1。

---

**第五步：告知并询问**

```
已登记 FINC-20260520-002「冲动下单难以自控」。

是否现在深入分析这个痛点，给出解决方案？（是/否）
```

**用户回复：是**

---

**第六步：创建三层文件**

文件夹：`{{PAIN_DIR}}/痛点记录/FINC-002-冲动下单/`

| 文件 | 用途 |
|------|------|
| `FINC-①买前查-下单检查清单.md` | 强制摩擦清单：冷静期 24h、仓位上限、止损预设、非对称性确认 |
| `FINC-②下单时填-交易记录.md` | 每笔交易：触发原因、检查项是否通过、仓位、预期止损 |
| `FINC-③每周日做-投资复盘.md` | 周复盘：本周交易回顾、冲动 vs 计划比率、行为模式变化 |

更新登记本该条目"解决方案"字段，链接到 `FINC-①买前查-下单检查清单.md`。

在 `{{PAIN_DIR}}/02-解决方案库.md` FINC 域追加摘要：

```
**FINC-002 冲动下单**
- 问题模式：情绪高峰时绕过理性判断直接下单
- 根因：无强制摩擦机制，依赖意志力
- 解决路径：买前查检查清单（24h 冷静期 + 仓位上限 + 止损预设）
- 效果验证：连续 4 周冲动交易笔数归零
- 对外输出潜力：可整理为"散户下单 SOP"文章
```
