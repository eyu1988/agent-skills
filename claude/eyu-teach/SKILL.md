# eyu-teach

## 路径常量

- 框架文件：`{{TEACH_DIR}}/GROW-①学前查-技术老师傅每日学习框架.md`
- 课堂笔记目录：`{{TEACH_DIR}}/课堂笔记/`

## 执行步骤

### 第一步：读取框架 + 扫描目录（并行）

同时执行：
1. 读取 `{{TEACH_DIR}}/GROW-①学前查-技术老师傅每日学习框架.md`，确认用户请求的概念属于哪个域、当前优先级
2. 列出 `{{TEACH_DIR}}/课堂笔记/` 下所有 `.md` 文件，读取每个文件的 frontmatter（domain / concept / status / score）

输出一行进度概览：

```
📚 已传授 N 个概念：[概念A · 域 · score分] [概念B · 域 · ⏳待答] …
今日：[概念名] · [域]（新课 / 续讲）
```

- 若该概念已有文件 → 读取现有文件，跳到未完成部分（未答的题 / 未填的字段），不重复讲解（除非用户说「重新讲」）
- 若该概念是新的 → 执行第二步

### 第二步：新建概念文件（仅新课）

在 `{{TEACH_DIR}}/课堂笔记/` 下新建文件，命名：`{域缩写}-{概念名}.md`

域缩写对照：

| 域 | 缩写 |
|----|------|
| iOS 深度补强 | iOS深度 |
| AI 工程化 | AI工程化 |
| 前端通识 | 前端通识 |
| 后端通识 | 后端通识 |
| 系统设计 | 系统设计 |
| DevOps 通识 | DevOps |
| 数据与算法 | 算法 |

文件初始 frontmatter：

```markdown
---
domain: {域}
concept: {概念名}
date: {今日日期}
status: 讲解中
score: ~
---
```

### 第三步：讲解 + 同步落库

**同时** 向用户输出讲解，并将相同内容写入文件「## 讲解」部分。

讲解固定六段结构：

1. **一句话定义**：最精准的概念核心
2. **类比解释**：用 iOS / 日常场景打比方，让用户秒懂
3. **核心机制**：底层是怎么运作的（3-5 个要点，可含代码）
4. **实际用法**：什么场景下用，代码/伪码示例
5. **常见陷阱**：新手容易踩的坑
6. **与 iOS 的关联**：连接用户的 iOS 知识，构建记忆锚点

讲解写入后，更新 frontmatter `status` → `待答题`。

### 第四步：出题（对话交互，不写文件）

紧接讲解输出三道题：

- **题 1（理解）**：用自己的话解释这个概念
- **题 2（应用）**：给一个贴近用户实际场景的情境题
- **题 3（陷阱识别）**：给出有问题的代码或描述，让用户找出问题

等待用户作答。

### 第五步：评分 + 回填文件

用户作答后：

1. 在对话中逐题评分，指出盲点，给出正确解释
2. 将考题、用户回答、评分、盲点写入文件「## 考题」部分
3. 更新 frontmatter：
   - `status` → `已完成`（全部作答）或 `部分完成`
   - `score` → 综合分（如 `85`）
4. 填写文件「## 综合自测分」「## 薄弱点」字段

### 第六步：确认

输出一行确认：`已归档 → 课堂笔记/{文件名}`
